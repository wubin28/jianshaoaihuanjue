● 关于动态路由中的 slug

  什么是 slug？

  slug 是一个占位符名称，代表URL中的动态部分。方括号 [slug] 表示这是一个动态路由参数。

  src/app/[slug]/page.tsx  →  /任意值
  src/app/blog/[slug]/page.tsx  →  /blog/任意值

  例如：
  - /hello → slug = "hello"
  - /blog/my-first-post → slug = "my-first-post"

  slug 只是约定俗成的命名，你可以用任何名称如 [id]、[username]、[postId]。

  特点

  在页面组件中通过 params 获取值：
  export default async function Page({ params }: { params: { slug: string } }) {
    const { slug } = await params;
    return <div>当前slug: {slug}</div>;
  }

  优势

  - 灵活性高：一个文件处理无限多个URL
  - SEO友好：支持语义化URL（如 /blog/nextjs-tutorial）
  - 代码简洁：无需为每个页面创建单独文件

  劣势

  - 需要数据验证：slug可能是无效值，需处理404
  - 调试稍复杂：需追踪参数来源

  适用场景
  ┌──────────┬────────────────────────────────┐
  │   场景   │              示例              │
  ├──────────┼────────────────────────────────┤
  │ 博客文章 │ /blog/[slug] → /blog/my-post   │
  ├──────────┼────────────────────────────────┤
  │ 用户主页 │ /user/[username] → /user/john  │
  ├──────────┼────────────────────────────────┤
  │ 商品详情 │ /product/[id] → /product/12345 │
  └──────────┴────────────────────────────────┘
