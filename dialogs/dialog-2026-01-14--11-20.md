## Claude code with sonnet 4.5:

```
● SSG (Static Site Generation) 静态站点生成

  定义: 在构建时（npm run build）预先生成HTML页面，而非用户请求时生成。

  特点:
  - 页面在部署前就已经生成为静态HTML
  - 所有用户看到的是相同的预渲染内容

  优势:
  - 加载速度极快（直接返回HTML，无需服务器计算）
  - 可部署到CDN，全球加速
  - 服务器压力小

  劣势:
  - 内容更新需要重新构建和部署
  - 不适合频繁变化的数据

  适用场景: 博客、文档站点、营销页面等内容变化不频繁的页面

  ---
  ISR (Incremental Static Regeneration) 增量静态再生

  定义: SSG的升级版——静态页面可以在后台按设定时间间隔自动更新，无需完整重新构建。

  特点:
  - 首次访问返回静态页面
  - 后台按设定时间（如60秒）检查是否需要重新生成
  - 下次访问时返回新版本

  配置示例:
  fetch('https://api.example.com/posts', {
    next: { revalidate: 60 } // 每60秒重新验证
  });

  优势:
  - 兼具SSG的速度优势
  - 无需完整重新部署即可更新内容
  - 适合大型站点（只更新需要更新的页面）

  劣势:
  - 用户可能短暂看到旧内容（在重新验证期间）
  - 配置比纯SSG稍复杂

  适用场景: 电商产品页、新闻列表、内容需要定期更新但不需要实时的页面

  ---
  简单对比:
  ┌──────┬──────────────┬────────────┐
  │ 方式 │   更新频率   │  典型用途  │
  ├──────┼──────────────┼────────────┤
  │ SSG  │ 仅构建时     │ 文档、博客 │
  ├──────┼──────────────┼────────────┤
  │ ISR  │ 定时后台更新 │ 电商、新闻 │
  └──────┴──────────────┴────────────┘
  ```
