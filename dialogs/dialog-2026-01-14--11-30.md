```
● async/await 简明解释

  async

  定义: async 是函数声明的关键字，表示这个函数是"异步函数"。

  // 普通函数
  function getData() { ... }

  // 异步函数
  async function getData() { ... }

  特点:
  - 异步函数总是返回一个 Promise 对象
  - 在异步函数内部可以使用 await

  ---
  await

  定义: await 用于等待一个异步操作完成，只能在 async 函数内部使用。

  async function getPosts() {
    const res = await fetch('https://api.example.com/posts'); // 等待请求完成
    const data = await res.json(); // 等待解析完成
    return data;
  }

  特点:
  - 暂停代码执行，直到 Promise 完成
  - 让异步代码看起来像同步代码

  ---
  对比：有 vs 没有 async/await

  // 没有 async/await（回调地狱）
  fetch('https://api.com/posts')
    .then(res => res.json())
    .then(data => console.log(data))
    .catch(err => console.error(err));

  // 有 async/await（清晰直观）
  async function getData() {
    const res = await fetch('https://api.com/posts');
    const data = await res.json();
    console.log(data);
  }

  ---
  优势

  - 代码可读性好: 异步代码像同步代码一样顺序书写
  - 错误处理简单: 可以用 try/catch 捕获错误
  - 调试方便: 可以像普通代码一样逐行调试

  劣势

  - 必须配对使用: await 只能在 async 函数中使用
  - 可能阻塞: 多个独立请求如果顺序 await 会变慢（应该用 Promise.all）

  适用场景

  - 网络请求 (fetch API调用)
  - 数据库查询
  - 文件读写
  - 任何需要等待结果的操作
```
