## Claude code with sonnet 4.5:

```
路径别名 @/* 解释

  在哪里使用？

  在项目的任何 .tsx 或 .ts 文件的 import 语句中使用。

  为什么需要它？

  解决深层嵌套导入问题。

  假设你的项目结构是这样的：
  src/
  ├── app/
  │   └── blog/
  │       └── [slug]/
  │           └── page.tsx    ← 你在这里
  ├── components/
  │   └── Button.tsx          ← 想导入这个
  └── utils/
      └── format.ts           ← 还有这个

  不使用别名（相对路径）：
  // src/app/blog/[slug]/page.tsx
  import Button from "../../../components/Button";  // 很丑，要数几个 ../
  import { formatDate } from "../../../utils/format";

  使用别名：
  // src/app/blog/[slug]/page.tsx
  import Button from "@/components/Button";  // 清晰，不管文件在哪层
  import { formatDate } from "@/utils/format";

  好处

  1. 路径简洁 - 不用数 ../
  2. 移动文件时不易出错 - 相对路径会因文件位置变化而失效
  3. 一眼看出是项目内文件 - @/ 开头的一定是 src/ 下的文件
```
