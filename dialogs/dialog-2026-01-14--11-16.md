```
● API路由 (API Routes)

  定义

  API路由是Next.js内置的后端功能，让你在同一个项目中创建HTTP API端点，无需单独搭建后端服务器。

  在App Router中，通过在src/app/目录下创建route.ts文件来定义API：

  src/app/
  ├── api/
  │   └── users/
  │       └── route.ts    → 访问路径: /api/users

  代码示例

  // src/app/api/users/route.ts
  export async function GET() {
    return Response.json({ users: ['Alice', 'Bob'] });
  }

  export async function POST(request: Request) {
    const data = await request.json();
    return Response.json({ created: data });
  }

  特点

  - 使用标准Web API（Request/Response）
  - 支持所有HTTP方法（GET、POST、PUT、DELETE等）
  - 与页面共享同一个项目结构

  优势

  - 部署简单: 前后端一起部署，无需分别管理
  - 类型共享: 前后端可共用TypeScript类型
  - 开发高效: 无需配置CORS，同域请求

  劣势

  - 扩展性有限: 高并发场景可能需要独立后端
  - 功能受限: 复杂业务逻辑不如专业后端框架灵活

  适用场景

  - 简单的数据接口（表单提交、数据查询）
  - BFF层（Backend For Frontend）
  - 与第三方服务的代理转发
  - Webhook接收端点
```
